### Why the **parent wheel** is fine but the **child-bonus wheel** still misses

| Component                    | Pointer math                                            | Label transform                      | Behaviour now                                    |
| ---------------------------- | ------------------------------------------------------- | ------------------------------------ | ------------------------------------------------ |
| **Parent `DailyBonusWheel`** | `FULL_SPINS × 360 + 270 − midPoint + jitter`            | `rotate(180 …)` on south-side labels | Pointer lands mid-slice; numbers upright         |
| **Child `ChildBonusWheel`**  | `FULL_SPINS × 360 − midPoint + jitter` *(270° removed)* | **no** rotation on south-side labels | Pointer lands on divider; bottom labels mirrored |

The child version dropped the **+ 270° offset** and the 180° flip.
Bringing it back to parity with the parent component solves both symptoms.

## 2-line geometry fix (child wheel)

```diff
// 🚩 inside spinMutation.onSuccess
- const midPoint = serverSegmentIndex * SEGMENT_ANGLE - 90 + SEGMENT_ANGLE / 2;
- const finalRotation = FULL_SPINS * 360 - midPoint + jitter;
+ const midPoint      = serverSegmentIndex * SEGMENT_ANGLE - 90 + SEGMENT_ANGLE / 2;
+ const finalRotation = FULL_SPINS * 360 + 270 - midPoint + jitter;  // ← add 270°
```

```diff
// 🚩 inside the <svg> map
```
